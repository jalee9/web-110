## Why This Lab Mattered

This lab was not just about adding more text to a webpage.

You practiced several foundational web development skills that will continue appearing throughout the course.

In Module 1, you created a simple webpage.

In Module 2, you expanded that same page instead of starting over.

This mirrors real web development. Developers rarely build something once and never revisit it. Instead, websites are updated, improved, reorganized, and expanded over time.

You are already beginning to think like a developer by improving an existing project.

### 2. HTML Structure and Organization

One of the biggest goals of this lab was learning how to organize content.

You practiced using:

- Headings (`<h1>`, `<h2>`)
- Paragraphs (`<p>`)
- Lists (`<ul>`, `<ol>`)

These elements help create structure.

Think of HTML as organizing information into meaningful sections rather than simply placing text on a screen.

A webpage with good structure is easier to read, easier to navigate, and easier to maintain.

This module introduced an important mindset:

> Choose HTML based on meaning, not appearance.

For example:

- Headings organize sections
- Paragraphs explain ideas
- Lists group related information

At this point in the course, HTML is not about making things look pretty.

Later, CSS will control colors, fonts, spacing, and layout.

Right now, you are learning how to create a strong foundation.

## Key Concepts to Remember

Here are the most important ideas from Module 2.

### HTML Structure

Most HTML elements follow this pattern:

```html
<element>Content goes here</element>
```

Many HTML elements use:

- An opening tag
- Content
- A closing tag

---

### Headings Create Organization

Headings create structure for your webpage.

Example:

```html
<h1>My Web Development Journey</h1>
<h2>Skills I Want to Learn</h2>
```

Remember:

- Use **one `<h1>`** for the main topic
- Use `<h2>` for major sections

Do not choose headings based only on size.

---

### Paragraphs Add Details

Paragraphs explain ideas.

Example:

```html
<p>
    I am learning web development because I want to understand
    how websites are created.
</p>
```

A heading introduces the topic.

A paragraph explains the topic.

---

### Lists Organize Information

Use unordered lists when order does not matter:

```html
<ul>
    <li>HTML</li>
    <li>CSS</li>
    <li>JavaScript</li>
</ul>
```

Use ordered lists when sequence matters:

```html
<ol>
    <li>Open VS Code</li>
    <li>Edit my webpage</li>
    <li>Preview my webpage</li>
</ol>
```

---

### Comments Help Organize Code

Comments are notes for developers.

Example:

```html
<!-- Skills section -->
```

Comments do not display on the webpage.

They help organize code and make projects easier to understand.

---

### Indentation Improves Readability

This:

```html
<ul>
<li>HTML</li>
<li>CSS</li>
</ul>
```

Works.

But this:

```html
<ul>
    <li>HTML</li>
    <li>CSS</li>
</ul>
```

Is easier to read.

Clean code matters.

---

## Common Beginner Mistakes

If you struggled with any of these, that is completely normal.

### Forgetting Closing Tags

Incorrect:

```html
<p>This is my paragraph.
```

Correct:

```html
<p>This is my paragraph.</p>
```

---

### Using Headings Based on Size

Incorrect thinking:

> “I want smaller text, so I’ll use H4.”

Better thinking:

> “I want to organize this section, so I’ll use an H2.”

Remember: HTML describes meaning.

CSS controls appearance later.

---

### Forgetting to Nest List Items Properly

Incorrect:

```html
<li>HTML</li>
<li>CSS</li>
```

Correct:

```html
<ul>
    <li>HTML</li>
    <li>CSS</li>
</ul>
```

---

### Skipping Browser Preview

Sometimes students only look at the code and never preview the webpage.

Always preview your work.

Browsers help you catch mistakes early.

---

### Confusing VS Code and GitHub

A common misunderstanding:

> “I saved the file in VS Code, so GitHub updated automatically.”

Not yet.

At this stage of the course:

1. Edit in VS Code  
2. Save the file  
3. Preview in the browser  
4. Update GitHub
