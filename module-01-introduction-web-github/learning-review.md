# Module 1 Learning Review

## Why This Lab Mattered

This lab introduced the basic workflow used throughout WEB 110.

A webpage begins as a file. In this module, students created an `index.html` file, tested it in a browser, and saved it in a GitHub repository.

This workflow will continue throughout the course as projects become more detailed.

## Key Concepts

### HTML Gives a Webpage Structure

HTML is used to organize webpage content. In this lab, the page included a heading and paragraph elements.

### The `index.html` File Matters

The file name `index.html` is commonly used for the homepage of a website. Using this file name helps build habits that will matter later when publishing webpages.

### The Browser Displays the Page

After creating the file, students opened it in a browser to make sure the page worked. Testing is an important part of web development.

### GitHub Stores the Project

GitHub was used to store the project online. In this module, students only needed to create a repository and upload files. More advanced GitHub skills will be added later.

## Common Mistakes to Watch For

- Naming the file something other than `index.html`
- Forgetting closing tags
- Placing content outside the `body` section
- Uploading the file inside an unnecessary folder
- Submitting a GitHub profile link instead of the specific repository link

## Keep Practicing

Before moving forward, students should be comfortable creating a simple HTML file, opening it in a browser, and finding the file inside a GitHub repository.
